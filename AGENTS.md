# AGENTS.md

This file provides guidance for AI agents and tools working in this repository.

## Project overview

SSB Designsystem is the design system for Statistics Norway (SSB).
It is built on top of [Digdir's Designsystemet](https://github.com/digdir/designsystemet) and adds SSB-specific tokens, CSS, and React components.

The repository is a **pnpm monorepo** with two public packages and one internal Storybook app.

---

## Repository structure

```
design-tokens/               # Generated token output — see token workflow below
designsystemet.config.json   # Token source config (colors, themes)
packages/
  css/                       # @statisticsnorway/design-css
  react/                     # @statisticsnorway/design-react
apps/
  storybook/                 # Internal Storybook (not published)
```

## Key commands

```bash
pnpm install              # Install all dependencies
pnpm build                # Build both packages (css then react)
pnpm test                 # Run React package tests
pnpm storybook            # Start Storybook dev server
pnpm tokens:create        # Regenerate design-tokens/ from designsystemet.config.json
pnpm tokens:inject        # Inject SSB token overrides between create and build
pnpm tokens:build         # Rebuild CSS theme files from existing token sources
pnpm tokens:update        # Run full token pipeline (create -> inject -> build)
pnpm format               # Format all files with Prettier
```

## Important conventions

### Generated vs source files

- `design-tokens/` is generated by `pnpm tokens:create` from `designsystemet.config.json`.
- `scripts/inject-token-overrides.ts` updates generated token files and is run via `pnpm tokens:inject`.
- `packages/css/theme/` is also generated — run `pnpm tokens:build` to update.
- Source of truth for colors and themes is `designsystemet.config.json`.

### Token workflow

- Run token steps in this order: `pnpm tokens:create` -> `pnpm tokens:inject` -> `pnpm tokens:build`.
- Use `pnpm tokens:update` to run the full token pipeline in one command.
- Use `pnpm tokens:build` only when you need to regenerate CSS theme files from existing token sources and no new token creation/injection is needed.

### Testing

- All custom components in `packages/react/src/components/` must have a co-located test file named `ComponentName.test.tsx`.
- Tests are not required for re-exported Digdir components.
- Run `pnpm test` to verify tests pass before release.

### Versioning and releases

Packages are versioned independently using [Changesets](https://github.com/changesets/changesets).
See [RELEASING.md](./RELEASING.md) for the full release flow.

## Development principles

- Prefer Digdir patterns and components over custom implementations unless there is a clear SSB-specific need.
- Keep public APIs small, consistent, and predictable across both CSS and React packages.
- Prioritize accessibility and semantic HTML, and align with existing Digdir naming conventions.
- Treat generated files as build artifacts: edit source files and scripts, then regenerate outputs.
- Minimize breaking changes; when unavoidable, document migration impact and include a Changeset.
- Keep implementation practical and maintainable; avoid premature abstraction.
- Validate changes in Storybook and tests before release.
- Update Storybook examples and documentation when public APIs change.
